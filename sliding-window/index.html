What Is a Sliding Window Protocol?

A sliding window protocol allows multiple frames to be sent before receiving acknowledgements. This improves network utilization by keeping the transmission channel busy rather than waiting for a response after every frame.

The sender maintains a transmission window while the receiver maintains a receive window. Frames inside the window may be transmitted and acknowledged independently.

What Are Duplicate ACKs?

Duplicate acknowledgements occur when the receiver continues receiving out-of-order frames while one earlier frame is missing. The receiver repeatedly acknowledges the last correctly received frame.

Modern TCP implementations use duplicate ACKs to detect packet loss before a timeout occurs.

What Is Fast Retransmit?

Fast retransmit allows the sender to retransmit a missing frame after receiving multiple duplicate acknowledgements rather than waiting for a retransmission timeout.

This significantly reduces recovery time on high-latency networks.

What Is Selective Acknowledgement (SACK)?

SACK allows a receiver to inform the sender about specific frames that arrived successfully even when earlier frames are missing.

This reduces unnecessary retransmissions and improves throughput.
